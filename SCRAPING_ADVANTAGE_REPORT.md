# スクレイピング限定データの活用による競争優位性の追求

YouTube Data API v3 は便利ですが、クォータ制限や情報の欠落が多く、競合ツール（APIのみを使用するもの）との差別化が困難です。
本プロジェクトにおいて、スクレイピング（`yt-dlp` やカスタムスクレイパー）でしか取得できない「競争優位性」を生むデータを以下に列挙します。

## 1. 関連動画（サイドバー）のデータ
**【APIでは取得不可】**
YouTube Data API の「関連動画取得（`relatedToVideoId`）」機能は2023年に廃止されました。
*   **価値**: どの動画のサイドバーに自社の動画が表示されているか、逆に自社の動画からどこに流出しているかを把握できます。「トラフィックの源泉」を特定し、コラボ先やメタデータ（タグ・タイトル）の最適化に直結します。

## 2. 検索結果のランキング（SERP）
**【APIでは実態と乖離】**
API経由の検索結果は、実際のユーザー（シークレットモード）が目にする検索結果や順位と異なる場合が多いです。
*   **価値**: 特定のキーワード（例：「ラランド」「お笑い 漫才」）での検索順位を時系列で追跡することで、VSEO（YouTube検索最適化）の効果を正確に測定できます。

## 3. 「コミュニティ」タブの投稿とエンゲージメント
**【APIでは取得不可】**
チャンネルの「コミュニティ」タブ（アンケート、画像投稿、テキストメッセージ）に関するデータは、現在のAPIでは一切提供されていません。
*   **価値**: 動画以外のファンとの接点、アンケート結果、それに対するコメント数やリアクションを分析することで、ファンコミュニティの熱量を定量化できます。

## 4. 動画内の「セグメント（チャプター）」の利用実態
**【APIでは概要欄のパースが必要、UI上の挙動は取得不可】**
APIでは説明文内のタイムスタンプは取得できますが、YouTubeが自動生成したチャプターや、UI上で強調されている区間は取得が困難です。
*   **価値**: どのシーンが「最も再生されている（Most replayed）」か、どの区間が視聴維持率に寄与しているかを外部から推測するヒントになります。

## 5. コメントの詳細ステータス（固定・ハート）
**【APIでは一部制限あり】**
クリエイターによる「固定コメント」や「ハート（いいね）」などのステータスは、APIの基本統計には含まれません。
*   **価値**: クリエイターがどの視聴者層を重視しているか、どのコメントを模範としているかの戦略が見えます。

## 6. プレミア公開・ライブ配信の同時接続数推移
**【APIではアーカイブ不可】**
`concurrentViewers` は配信中しか取得できず、終了後は消えてしまいます（APIの履歴には残りません）。
*   **価値**: 配信中の盛り上がりのピークや、どのタイミングで視聴者が離脱・流入したかを「秒単位」で記録し、後から分析できます。

## 7. 広告の有無と種類
**【APIでは取得不可】**
どの動画に広告が付いているか、どのような種類の広告が流れているかはAPIから取得できません。
*   **価値**: ライバルの動画の収益化状況や、どのような企業の広告が自分たちの動画に関連付けられているか（ブランドの親和性）を確認できます。

## まとめ：次のステップへの提案

本プロジェクトの `scrape_collector.py` を拡張し、まずは**「1. 関連動画」**および**「2. 検索順位」**の取得を実装することで、既存のAPIベースのツールでは実現できない「視聴者の流入経路分析」が可能になると考えられます。
